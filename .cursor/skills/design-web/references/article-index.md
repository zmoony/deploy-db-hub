**系列顺序（旧→新）：** 工作台 → 呼吸感 → 财务SaaS

### 文章 1 要点 — 工作台

- 核心心法：**信息降噪** — 把复杂数据用最轻量视觉语言翻译出来。
- **模块化卡片布局：** 留白 + 低饱和背景（青绿、灰蓝、浅紫）划分模块（数据总览、图表、列表）。
- **图形叙事：** 用面积图/折线图/圆环图讲趋势，而非堆砌数字。
- **核心 KPI 优先：** Top Card 放左上角；大字号 + 绿涨红跌百分比，1 秒判断业务健康度。
- 注入品牌温度，但保持中性色底 + 状态色点缀。

### 文章 2 要点 — 呼吸感

- SaaS 最怕信息过载，"闷"且"挤"。
- **克制视觉语言：** 大面积留白 + 清透柔和渐变 → 呼吸感、高级感。
- **极简卡片网格 + 彩色状态标签**（已发布、审核中）→ 降低认知负荷。
- **做减法比做加法难；** 不堆颜色元素，干净样式传达高效功能。

### 文章 3 要点 — 财务 SaaS

- 相对文章 2 再减法：取消复杂渐变，色彩大幅克制。
- **色彩是外衣，信息是内核；** 后端用户讨厌"美颜滤镜"，要清晰准确的数据逻辑。
- **红涨绿跌** 精准传达，但不满屏高饱和色块。
- **好看、好用，但不喧宾夺主；** 1 秒找到该赚多少钱、哪笔账单超支。
- 听劝务实，业务场景第一位；忌设计师自我感动。

### 文章 4 要点 — 地图组件

- 同一业务（世界地图）质感差异来自：底图风格、标注密度、配色温度、交互层次。
- 避免"默认灰地图 + 粗标注"；用适度饱和区域色 + 细线边界 + 分层 legend。
- 信息分层：国家/区域 → 数值 → 交互 tooltip；不要一次展示全部标签。
- 与 B 端地图选型：静态展示用简化 GeoJSON + 柔和 choropleth；实时追踪才用深色底 + 高对比 marker。

### 文章 5 要点 — 飞控大屏操控感

- **操控感 = 实时反馈 + 物理隐喻 + 低延迟视觉链。**
- 布局：左/中地图航迹 + 右仪表/参数面板；或顶栏状态 + 全屏地图 + 浮层 HUD。
- 核心组件：姿态仪、高度/速度仪表、航迹地图、告警条、设备列表。
- 视觉：深色底（`#0B1220` ~ `#1a2332`）+ 高亮青/蓝/绿数据；数字等宽字体。
- 数字需大且稳定；关键告警用红/黄闪烁或边框，非全屏色块。
- 地图 marker 带序号与时间轴联动；选中高亮 + 轨迹线。
- 避免装饰性 3D 影响读数；动画仅用于状态变化（blink on alert）。

### 文章 6–7 要点 — AI 科技视觉 / PPT

- 手搓布局 + AI 辅助出图：先定构图与层次，再用 AI 生成背景/纹理/插画。
- 科技感关键词：深色底、霓虹渐变边、细线框、粒子/网格纹理（低 opacity）。
- 控制 **最多 1 个发光焦点**（标题区或主视觉），其余区域保持暗色留白。
- PPT/落地页与 B 端后台不同：可更高饱和，但信息区仍保持可读对比度。
- 工作流程：参考收集 → 网格构图 → AI 背景 → 手工叠 UI 框/文字 → 统一色温。
